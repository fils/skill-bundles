## Emerging patterns

1. **Academic convergence on skill formalization** — three simultaneous surveys (SoK, 4-axis, 6-layer) all converge on: skills = multi-file packages ≠ tools; lifecycle model needed; security is critical. The S=(C,π,T,R) definition from SoK is becoming the canonical formal vocabulary.
2. **Self-evolution + co-evolutionary verification** — CoEvoSkills introduces the verifier as a co-evolving context element, not just a static gate. This generalizes the [[three-layer-validation-stack]] pattern from static to dynamic verification.
3. **Supply-chain provenance for skills** — the SLSA/in-toto extension to agentic code generation (Konflux/fullsend) directly applies to skill marketplace provenance. The "intent → source" gap maps to skill authoring provenance.
4. **Marketplace risk quantified** — ClawHavoc's 13% malicious rate + SoK's 26.1% vulnerability rate + arXiv:2602.08004's 37% flaw rate all converge: **roughly 1 in 4–8 marketplace skills is malicious or vulnerable**.
5. **Dependency-aware retrieval validated** — GoS v3 provides concrete benchmark evidence that dependency-aware skill bundle retrieval outperforms flat retrieval in both quality (+25.55%) and efficiency (-56.72% tokens).
