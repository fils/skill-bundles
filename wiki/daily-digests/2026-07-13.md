## Emerging patterns

1. **Skill optimization taxonomy completing** — Four distinct approaches now mapped: SkillMOO (search/NSGA-II), CoEvoSkills (evolution), SkillReducer (content reduction), and now SkillOpt (training/gradient-like text optimization) + EvoSkill (failure-driven discovery). Each addresses a different axis. SkillOpt outperforms all others on 52/52 cells.
2. **Skill transfer is real and quantified** — EvoSkill (zero-shot task→task transfer +5.3%), SkillOpt (cross-harness Codex→Claude Code +59.7, cross-model-scale), SkillMigrator (cross-domain 35.4% reuse). Skills capture reusable workflow knowledge, not benchmark-specific instructions. Transfer is the key evidence for the skill bundle thesis.
3. **Skill generation is the bottleneck** — SkillGenBench proves generating skills (7-27% pass@3) is 2-3x harder than using them. SkillsBench shows curated skills help; SkillGenBench shows auto-generating them fails often. Negative transfer is a real risk. This shifts the research frontier from skill use to skill creation.
4. **Skills need OS-level management** — Skill OS paper argues skills are applications requiring OS abstractions (caching, env construction, global management, failure handling, security). The 100K skill analysis provides the evidence base. ORCA is an early implementation; Skill OS is the theoretical framework.
5. **Provenance standards converging** — PROV-AGENT extends W3C PROV + MCP for agent provenance; W3C Agent Protocol Community Group active; SLSA/in-toto for supply chain. Provenance is becoming a standard context element for skill bundles.
6. **Cross-domain via structural layout** — SkillMigrator's TIP abstraction (accessibility-tree skeleton + slot schema) enables cross-domain skill transfer by matching layout structure, not element IDs. This is a novel architectural pattern for skill bundles.
