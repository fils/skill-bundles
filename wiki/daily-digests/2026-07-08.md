# Daily Digest: 2026-07-08 — Skill Bundles Research

**Sources ingested (6):** agentskills.io specification & home, SHACL 1.2 Core (W3C), SSSOM ontology mappings, VS Code Agent Skills docs, addyosmani/agent-skills (72.5k stars), anthropics/skills (159k stars).

**Key discoveries & patterns:**
- **Agent Skills standard (agentskills.io)**: Core packaging format (SKILL.md + optional scripts/references/assets). Progressive disclosure for efficiency. Open standard adopted across Claude, Codex, Copilot, Gemini, etc. Direct support in VS Code for project/personal skills.
- **SHACL 1.2**: Formal validation + new agent-specific primitives (sh:agentInstruction, sh:intent). Used for guardrails, policy compliance, ontology generation for agents. Strong complementarity with skills (validation layer).
- **SSSOM**: Semantic mapping standard for ontology alignment, with rich metadata (confidence, justification, curation rules, provenance). Enables interoperable skill bundles across domains.
- **Large catalogs**: addyosmani (engineering focus, evals, multi-agent support) and anthropics (Claude demos, API skills) show real-world bundle scale. Often include or can be extended with SHACL/SSSOM context.

**Emerging patterns:**
- Bundles frequently combine skills with validation (SHACL) and semantic layers (SSSOM) for reliable agent composition.
- Cross-platform portability emphasized; skills as reusable "context artifacts" + formal specs.

**New research questions:** How many documented bundles explicitly pair SKILL.md with SHACL shapes or SSSOM mappings? Gap in ontology/taxonomy coverage for agent skills.
