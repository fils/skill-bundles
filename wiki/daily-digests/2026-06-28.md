# Daily Digest — 2026-06-28 (Iteration 28)

## Key Signals

### 1. SkillFortify: Formal Verification for Agent Skills (arXiv:2603.00195)
- **First formal analysis framework** for the agentic AI skill supply chain.
- Adapts the **Dolev-Yao attacker model** to the 5-phase skill lifecycle (Install→Load→Configure→Execute→Persist).
- Uses **abstract interpretation** over a 4-element capability lattice: `{NONE ⊑ READ ⊑ WRITE ⊑ ADMIN}`.
- **Soundness guarantee:** If analysis reports safe, concrete execution *cannot* exceed declared capabilities.
- **Performance:** 96.95% F1, 100% precision (zero false positives), 2.55ms/skill scan, 1k-node dependency resolution in 92.4ms.
- **Artifacts:** Generates `skill-lock.json` (deterministic lockfile with hashes + trust scores) and **ASBOM** (CycloneDX 1.6-compatible Agent Skill Bill of Materials).
- **SAT-based dependency resolution** via Glucose3 solver: ensures version constraints met, no conflicts, security bounds enforced across skill chains.
- **Trust Score Algebra:** Multi-signal (Provenance, Behavioral, Community, Historical) with monotonicity property and ~139-day half-life exponential decay.
- Supports Claude Code Skills (MD+YAML), MCP (JSON-RPC), and OpenClaw Skills (YAML manifests).

### 2. AgentVerify: Compositional Formal Verification via LTL (Preprints.org, April 2026)
- Treats LLM as **non-deterministic oracle** within a verifiable **Finite-State Machine (FSM)**.
- **Two-tier hybrid architecture:** (1) O(1) runtime monitor with "obligation tokens" for immediate intervention; (2) Offline post-hoc analyzer using **Büchi automaton** construction and product model checking.
- **23 LTL specification templates** across 4 domains: Memory Integrity, Tool Call Safety, MCP/Skill Invocation, Human Interaction Boundaries.
- **Results:** 86.67% accuracy, 100% catastrophic failure detection rate, only 0.04% latency overhead per step.
- **NL-to-LTL generator:** Translates natural language safety requirements into formal logic with 87% accuracy.
- Regulatory readiness: Structured reports with counterexample traces for **EU AI Act** and **NIST AI RMF** compliance.
- Neural verification fails (BERT-based: 13.33%); temporal logic essential for multi-step violation patterns.

### 3. SSL Representation: From Skill Text to Skill Structure (arXiv:2604.24026, Peking University)
- **First structured representation** for agent skills: Scheduling-Structural-Logical (SSL) three-layer JSON graph.
- Draws on **Schank & Abelson** classical linguistic knowledge representation.
- **Three layers:** (1) Scheduling (when to invoke: goals, intent signatures, control-flow); (2) Structural (how work is organized: scenes like Prepare, Acquire, Act, Verify); (3) Logical (what resources: atomic actions READ/CALL_TOOL with resource scopes CREDENTIALS/NETWORK).
- **Restricted vocabularies (enums):** Scene Types (PREPARE, ACQUIRE, REASON, ACT, VERIFY, RECOVER, FINALIZE), Logical Primitives (READ, SELECT, COMPARE, VALIDATE, INFER, WRITE, CALL_TOOL, REQUEST), Resource Scopes (MEMORY, LOCAL_FS, CODEBASE, PROCESS, USER_DATA, CREDENTIALS, NETWORK).
- **Normalizer:** LLM-based 4-pass pipeline (Skill Record Extraction → Scene Decomposition → Logic-Step Expansion → Verification).
- **Evaluation I (Skill Discovery):** 403 queries vs 6,184 candidates. Baseline 0.573 MRR → SSL-Rich **0.707 MRR** (+23.4%).
- **Evaluation II (Risk Assessment):** 500 skills. Baseline 0.744 Macro F1 → +SSL **0.787 Macro F1** (+5.8%).
- Key insight: "Don't Replace, Augment" — SSL is an evidence interface; keep original SKILL.md for nuanced interpretation.

### 4. Comprehensive Survey on Agent Skills (SSRN 6656500, CUHK, April 2026)
- Defines agent skills as **"reusable procedural artifacts that coordinate tools, memory, and runtime context under task-specific constraints."**
- **4-stage lifecycle:** (I) Representation → (II) Acquisition → (III) Retrieval → (IV) Evolution.
- Agents handle high-level reasoning; skills serve as the **operational layer** for reliable, reusable, composable execution.
- Key challenges: Quality control (validating synthesized skills), Interoperability (cross-architecture sharing), Safe updating (managing breaking changes), Long-term capability management (organizing vast skill libraries).
- Impact: Roadmap from "one-off" agent prompts toward **persistent agentic systems**.

- **Scale:** SkillsMP indexes **1.9 million** public skills scraped from GitHub.

## Patterns Observed

### Pattern 1: Formal Verification is the New Validation Layer
SkillFortify (abstract interpretation + SAT-based dependency resolution) and AgentVerify (LTL model checking) represent a **paradigm shift from heuristic scanning to formal mathematical guarantees**. This parallels the SHACL validation pattern but goes further: instead of validating data shapes, these frameworks formally verify that agent skills cannot exceed their declared capabilities. The capability lattice `{NONE ⊑ READ ⊑ WRITE ⊑ ADMIN}` is the skill-level equivalent of SHACL's constraint shapes.

### Pattern 2: Structured Skill Representations Enable Scale
The SSL representation (Peking University) demonstrates that converting unstructured SKILL.md text into a typed three-layer graph improves both skill discovery (+23.4% MRR) and risk assessment (+5.8% F1). This is evidence that the **representation layer** of the agent skill lifecycle (as identified in the CUHK survey) is a critical bottleneck. The restricted vocabularies (Scene Types, Logical Primitives, Resource Scopes) function like controlled vocabularies in ontology work — a direct parallel to the SSSOM mapping pattern.

### Pattern 3: The Quality-Security Dual Threat
The ecosystem report quantifies what was previously qualitative: 1.9M public skills with an average quality score of 6.2/12, and 36% containing prompt injections. The **quality gap** (6.2/12 average) and the **security gap** (36% prompt injection rate) are two sides of the same coin: the lack of formal context artifacts (validation, mapping, rules) bundled with skills. This is the strongest evidence yet that skill bundles — skills packaged with formal context elements — are not just an academic curiosity but a practical necessity.
