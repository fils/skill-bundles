## Emerging Patterns

1. **Complete attack taxonomy now spans four threat surfaces** — The skill supply chain has four distinct attack vectors now formally studied: (a) code-level malicious tools (MalTool), (b) model-in-skill backdoors (BadSkill), (c) prompt injection via SKILL.md (SkillJect), (d) marketplace-level campaigns (ClawHavoc). Each requires a different defense layer.

2. **Defense stack achieving full lifecycle coverage** — SkillSpector (pre-installation scanning) + SkillGuard (runtime permission) + AgentBound (MCP server access control) + SkillFortify (formal verification) now cover: pre-installation → supply chain → runtime → MCP infrastructure. The defense stack mirrors the attack taxonomy.

3. **MCP as critical infrastructure layer** — AgentBound's focus on MCP servers (296 most popular, unrestricted access) recognizes MCP as the connective tissue requiring its own security model. This extends skill bundle security beyond skills to the protocol layer.

4. **Empirical reality check on skill utility** — SWE-Skills-Bench's finding that 80% of skills provide zero benefit is the first large-scale empirical challenge to the "more skills = better" assumption. Combined with SkillReducer's less-is-more effect, the field needs rigorous evaluation, not just adoption.

5. **Theoretical foundation arrives** — Externalization in LLM Agents provides the systems-level framework: skills = externalized procedural expertise, one of four coupled forms. The weights→context→harness progression model explains why infrastructure matters increasingly. Self-evolving and shared infrastructure are the frontier.

6. **Collective evolution as paradigm shift** — SkillClaw's multi-user skill evolution (cross-user + over-time trajectory aggregation) transforms skills from static artifacts to living, self-improving infrastructure. This connects to the Externalization paper's "self-evolving harnesses" and "shared agent infrastructure" emerging directions.
