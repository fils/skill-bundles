## Sources

1. **WildClawBench** (arXiv:2605.10912, InternLM, May 2026) — Native-runtime long-horizon benchmark: 60 bilingual multimodal tasks in real CLI harnesses (OpenClaw, Claude Code, Codex, Hermes); hybrid grading; 19 models, best 62.2% (Opus 4.7); harness switch shifts scores up to 18pp; skill ablation mixed but Code Intelligence +22.4 for strongest model; GitHub InternLM/WildClawBench (~477★)
2. **SKILL0** (arXiv:2604.02268, ZJU+Meituan+Tsinghua) — First RL framework for skill internalization ("skills at training, zero at inference"); Dynamic Curriculum anneals skill scaffolding by on-policy helpfulness; +9.7% ALFWorld, +6.6% Search-QA, +10.1% WebShop; <0.5k tokens/step; GitHub ZJU-REAL/SkillZero
3. **OpenClaw-Skill / CSTS** (arXiv:2606.16774) — Collective Skill Tree Search: multi-model CSN-Gen + CSN-Assess (quality + transferability scoring); builds structured transferable skill trees; Collective Skill RL multi-skill selection; addresses fragmentation, single-model bias, poor transfer
4. **R3-Skill** (arXiv:2606.03565 v4, Tencent) — Query-conditional skill routing with **skill compatibility**; Reject-as-Resource uses LLM rejections as co-retrieval supervision; 10,246 skills, 41k queries, 32k rejections; Hit@1=0.7521; HF R3-embedding/rerank 0.6b; GitHub Tencent/R3-Skill
5. **ASSCs / Skills Are Not Islands** (arXiv:2607.01136, PKU, Jul 1) — Agent Skill Supply Chains; SDA recovers multi-channel deps (skill/package/service); SkillBOM; **1.43M skills**; 58.73% name collisions; 71–73% packages transitive-only; 60–78% security skill deps transitive; recommends typed manifests + lockfiles
6. **From Registry to Repository** (arXiv:2607.00911, Jul 1) — First empirical skill-as-artefact study; 18,463 registry + 23,199 personal skills; 3,709 reuse links; 6 content themes + 6 maintenance themes; reuse = one-time copy; 53% never modified; behavioral contracts stable
7. **Inside the Skill Market** (arXiv:2607.09065, Jul 10) — First activity-centric SE skills study; 11,497 SE skills; SDLC coverage skewed to construction; versioning immature (most single-version); five-component skill model; evaluation gap for reuse effectiveness

## Emerging Patterns

1. **Internalization vs externalization spectrum completes** — SKILL0 (weights absorb skills) + Externalization review (skills as cognitive artifacts) + Memento/SkillOpt (external editable) now form a clear design triangle: train-away skills · keep skills as memory · optimize skills as text parameters.
2. **Composition-aware retrieval is mandatory infrastructure** — R3-Skill proves independent relevance fails; SkillsBench/SWE-Skills-Bench negative deltas are partly a routing/compatibility problem, not only content quality.
3. **Supply chain moves from incident to inventory science** — ASSCs/SkillBOM on 1.43M skills operationalizes SBOM+lockfile for skills; transitive dark matter (packages + risks) is the dominant surface.
4. **Native multi-harness evaluation standard** — WildClawBench shows harness choice can matter as much as model choice (±18pp); skill utility must be measured in production runtimes (OpenClaw/Claude Code/Codex/Hermes).
5. **Maintenance science arrives** — Registry→Repository + Inside the Skill Market: skills are copy-fork artefacts with additive local evolution, stable behavioral contracts, and weak version maturity signals.
6. **Collective construction ≠ collective evolution** — CSTS (multi-model tree construction + transfer gates) is orthogonal to SkillClaw (multi-user runtime trajectory aggregation); both “collective” but different lifecycle phases.
