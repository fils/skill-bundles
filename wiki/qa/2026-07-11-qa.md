### Q2: What is the "intent → source → binary" gap and how does it apply to skill bundles?

The supply chain is now `intent → source → binary` (per [supply chain agentic factory in toto](../examples/supply-chain-agentic-factory-in-toto.md)):
- **source → binary:** Well-secured via SLSA, in-toto, hermetic builds, SBOMs
- **intent → source:** Largely unexamined — AI agents generate code from human intent, but we have no provenance for this transformation

**For skill bundles:** Skills are authored by humans (intent → skill) and increasingly by AI agents (prompt → skill). Without provenance for the "prompt → skill" step:
- A compromised model could generate malicious skills (ClawHavoc pattern)
- Human-machine cognitive misalignment could produce subtly broken skills (CoEvoSkills finding)
- No attestation of which model, prompt, or review process produced a given skill

**Solution direction:** Extend in-toto attestations to cover skill generation; neutral observer pattern (sign from outside the generation process); policy gates that verify skill provenance before marketplace listing.

### Q3: How does CoEvoSkills' co-evolutionary verification relate to the existing three-layer validation stack?

The [three layer validation stack](../concepts/three-layer-validation-stack.md) is static:
- Layer 1 (pre-commit): skill-validator CLI
- Layer 2 (CI/CD): validate-skill GitHub Action
- Layer 3 (registry): agentskill.sh security scoring

CoEvoSkills introduces **dynamic** verification:
- The Surrogate Verifier co-evolves with the Skill Generator
- It provides feedback without ground-truth test content
- It adapts as the generator improves

**Generalization:** Co-evolutionary verification is the adaptive/dynamic counterpart to the static three-layer stack. Future validation stacks could combine both: static pre-commit/CI/registry gates + dynamic co-evolutionary verification for skill generation.

## Gap Analysis

### Lifecycle Coverage Gaps

The SoK's 7-stage lifecycle model (discovery → practice → distillation → storage → composition → evaluation → update) reveals gaps:

- **Practice:** Few examples document how skills are rehearsed/refined before deployment
- **Distillation:** Limited coverage of how skills are compressed from raw experiences
- **Update:** Most examples document initial creation, few track skill evolution over time

### Context Element Gaps

Newly identified but underexplored context elements:
- **Co-evolutionary verification** — only 1 example (CoEvoSkills); needs more
- **SLSA/in-toto provenance** — only 1 example (Agentic Factory); needs more
- **Selective internalization** — mentioned in 6-layer survey but no wiki examples
- **Marketplace governance models** — SkillsMP/LobeHub at scale (425K/170K) not yet documented
