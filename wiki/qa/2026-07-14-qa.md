# Q&A — 2026-07-14 (Iteration 39)

## Q1: How does the security governance stack now cover the skill lifecycle?

The skill lifecycle security governance now has four layers:
1. **Publication** — NVIDIA-verified skills (catalog → scan → sign → document → distribute)
2. **Supply chain** — SkillFortify (formal verification: DY-Skill attacker model, static analysis, lockfile, trust scores)
3. **Runtime** — SkillGuard (dual-plane permission governance: context + action, deny-by-default)
4. **Detection** — SkillSieve (three-layer malicious skill triage, F1=0.920)

This is a complete security stack: from publication through supply chain through runtime to detection. SkillFortify and SkillGuard are the two key new additions that provide **formal guarantees** (not just heuristics) — SkillFortify via mathematical proofs (abstract interpretation, capability confinement), SkillGuard via systematic permission enforcement.

## Q3: What is the relationship between Memento-Skills' Read-Write cycle and Skill1's select-use-distill?

Both are skill lifecycle models but at different abstraction levels:
- **Memento-Skills (Read-Write):** Read = skill router selects relevant skill; Write = agent updates/expands skill library. This is a **two-phase** model focused on skill access and creation.
- **Skill1 (Select-Use-Distill):** Select = query + re-rank; Use = task solving; Distill = new skill from trajectory. This is a **three-phase** model that separates utilization from creation.

Skill1's key insight is that these three capabilities must co-evolve under a single reward signal (frequency decomposition). Memento-Skills achieves continual learning without RL (via memory-based adaptation). Both prove that skills can serve as the sole adaptation mechanism (no LLM parameter updates needed).

## Q5: How does SkillFortify's lockfile compare to traditional package management?

SkillFortify adapts npm/PyPI lockfile patterns to agent skills:
- **Deterministic ordering** (alphabetical, byte-identical output)
- **Dependency resolution** (constraint satisfaction via SAT solver)
- **Trust scores at lock time** (temporal comparison for degraded confidence)
- **Analysis status** (critical blocks, warning recorded)
- **SHA-256 integrity hash** (tamper detection)
- **Agent Dependency Graph** (SAT-based resolution for 1,000-node graphs in <100ms)

This is the first package-management-style approach for agent skills. The key difference from npm/PyPI: skills are semi-structured (natural language + code + data), so static analysis is more complex. SkillFortify uses abstract interpretation (formal method) rather than AST parsing.
