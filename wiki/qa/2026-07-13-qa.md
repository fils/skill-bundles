# Q&A — 2026-07-13 (Skill Bundles, Iteration 38)

## Q1: Which skill optimization approaches now exist, and how do they relate?

**Five distinct approaches are now mapped:**

| Approach | Method | Key Metric | Transfer |
|----------|--------|------------|----------|
| **SkillMOO** | NSGA-II Pareto search (pass rate + cost) | Top rank 11/12 SkillsBench | Not tested |
| **CoEvoSkills** | Evolutionary (generator + surrogate verifier) | Surpasses human-curated in 5 iterations | 8-LLM generalization |
| **SkillReducer** | Content reduction (delta debugging + progressive disclosure) | 48%+39% compression, +2.8% quality | 5 models, 4 families |
| **SkillOpt** | Training-style (forward-backward-update, textual LR, validation gating) | Best/tied-best on 52/52 cells, +23.5 on GPT-5.5 | Cross-model-scale, cross-harness, cross-task |
| **EvoSkill** | Failure-driven discovery (3-agent: Executor→Proposer→Skill-Builder) | +7.3% OfficeQA, +12.1% SealQA | Zero-shot to BrowseComp (+5.3%) |

**Relationship:** SkillOpt explicitly outperforms EvoSkill, GEPA, TextGrad, and others on all 52 cells. EvoSkill focuses on *discovery* (creating new skills), while SkillOpt focuses on *optimization* (improving existing skills). SkillReducer reduces content, CoEvoSkills co-evolves, SkillMOO searches configuration space. They address different axes and could potentially compose.

## Q2: What evidence exists that skills transfer across domains/models/tasks?

Three independent lines of evidence from today's sources:
1. **EvoSkill:** Skills evolved on SealQA transfer zero-shot to BrowseComp (+5.3%) without modification — proving skill-level optimization produces transferable capabilities beyond the training task.
2. **SkillOpt:** Cross-harness transfer (spreadsheet skill trained in Codex → used in Claude Code: 22.1 → 81.8, +59.7) outperforms direct training (80.4). This suggests skills capture general workflow logic, not harness-specific recipes. Also transfers across model scales (GPT-5.4-mini+skill > GPT-5.4 no-skill).
3. **SkillMigrator:** Cross-domain web skill reuse rate of 35.4% (vs PolySkill's 31%) on Mind2Web. Uses structural layout matching (accessibility-tree skeletons) rather than element references.

## Q3: Why is skill generation harder than skill use?

SkillGenBench provides the first systematic evidence:
- Repository-grounded skill generation: 7-16% pass@3 (procedures are implicit in code structure)
- Document-grounded skill generation: 23-27% pass@3 (procedures explicit but scattered)
- SkillsBench showed curated skills substantially improve performance
- But auto-generated skills can induce **negative transfer** (agent performs worse with the skill than without)
- The generator-executor decoupling isolates skill distillation quality from planning and retrieval confounds

The gap is 2-3x: using a well-crafted skill is relatively easy; distilling one from raw corpora is the bottleneck.

## Q4: What would a Skill OS need to provide?

Based on the 100K skill analysis (Skill OS paper), five demands:
1. **Locality-aware caching** — cache skill execution results across invocations (skills imply repeated reuse)
2. **Dynamically constructed environments** — build execution environments on demand (skills share runtime requirements)
3. **Global skill management** — cross-session visibility for optimization (currently skills shared without global management)
4. **Structured failure handling** — deterministic error handling, not trial-and-error (dependency failures cause token waste)
5. **Runtime security & auditing** — enforce parallel-safety, idempotency (currently prompt-level instructions inadequate)

ORCA (122 capabilities, 36 DAG skills, policy gates, provenance) is an early partial implementation of these demands.

## Q5: How does provenance apply to skill bundles?

PROV-AGENT extends W3C PROV + MCP to capture:
- Agent decisions, prompts, and responses as first-class provenance components
- End-to-end workflow provenance graphs (agent actions = first-class, like traditional workflow tasks)
- Cross-facility (edge, cloud, HPC) near real-time capture

This complements:
- SLSA/in-toto (supply chain provenance for skill generation)
- W3C Agent Protocol Community Group (interoperable agent protocols)
- The provenance tracking in ORCA (policy gates + provenance per execution)

Together, provenance is becoming a standard context element for skill bundles — tracing from skill definition through execution to downstream impact.
