## Complex Queries Against Wiki Catalog

### Q1: Which bundles combine skills with both dependency graphs AND validation?
**A:** [[graph-of-skills-dependency-retrieval]] provides dependency graph construction. [[dspy-agent-skills-bundle]] has 80 validation tests. No single bundle yet combines both dependency-aware retrieval + programmatic skill validation. This is a gap: a bundle that validates both the correctness of individual skills AND their dependency compatibility before loading.

### Q2: What benchmarking coverage exists for skill bundles?
**A:** [[skillsbench-agent-skills-benchmark]] provides paired evaluation (with-skills vs without-skills delta). [[awesome-agent-skills-catalogs]] catalogs the market but doesn't benchmark it. Gap: no per-bundle benchmark — SkillsBench measures aggregate augmentation, not individual bundle effectiveness.

### Q3: Which bundles encode security posture as context elements?
**A:** [[nvidia-verified-agent-skills]] (OpenSSF signing), [[mondoo-agent-skills-security]] (security analysis cataloging 14 attack patterns), [[veto-agent-authorization]] (authorization kernel). Mondoo's analysis identifies that 25%+ of public skills have vulnerabilities — verification bundles exist but coverage is thin for the 50k+ skills in registries.
