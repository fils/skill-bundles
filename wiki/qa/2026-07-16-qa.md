# Q&A — 2026-07-16 (Iteration 41)

## Q1: Do skill bundles help more when kept external, or when internalized into weights?

**A:** Both are now rigorously defined poles.

- **External (runtime packages):** SkillOpt treats SKILL.md as trainable text; Memento uses markdown skills as evolving memory; marketplace skills load on demand (agentskills.io).
- **Internalized (weights):** SKILL0's Dynamic Curriculum starts with full skill context and anneals to zero-shot, gaining +9.7% ALFWorld / +10.1% WebShop with <0.5k tokens/step.

**Trade-off:** External skills remain inspectable, versionable, and swappable (supply-chain + governance friendly; see ASSCs, NVIDIA-verified). Internalized skills remove retrieval noise and token tax but shift the sovereignty/adaptability boundary to training time. Epistemic check: SKILL0 abstract + Externalization four-form taxonomy + SkillOpt 52/52 cells — three independent lines.

**Confidence:** High (direct paper results).

## Q2: Why can "more skills" hurt agent performance?

**A:** Three complementary mechanisms now cataloged:

1. **Utility** — SWE-Skills-Bench: 39/49 skills zero gain; SkillsBench: 16/84 tasks negative delta.
2. **Compatibility** — R3-Skill: top-K fails when co-retrieved skills conflict under the query; independent relevance is insufficient.
3. **Supply-chain opacity** — ASSCs: most packages and many security signals are transitive-only; bloated graphs raise install and attack surface.

**Implication:** Bundle design needs retrieval with Set-Compat objectives, SkillBOM/lockfiles, and empirical paired evaluation (SkillsBench / WildClawBench skill ablations).

**Confidence:** High.
