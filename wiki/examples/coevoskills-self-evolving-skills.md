## Name & Origin

CoEvoSkills is the first framework to address the **tool–skill gap** in autonomous skill generation. While existing self-evolving methods work for tools (single functions), skills are structured bundles of interdependent multi-file artifacts (workflow instructions, scripts, domain references). CoEvoSkills couples a Skill Generator with a co-evolving Surrogate Verifier to autonomously construct complex skill packages without ground-truth test labels.

## Skills Included

- **Skill Generator:** Iteratively refines multi-file skill packages (SKILL.md + scripts/ + references/)
- **Surrogate Verifier:** Co-evolves with the generator to provide informative, actionable feedback
- **Skill-Creator Workflow:** Draft → test → iterate (≥3 iterations) → save to `.claude/skills/`
- **YAML Frontmatter:** Proper name + description per skill
- **Progressive Disclosure:** Constraints → workflow → edge cases → scripts/

## Context Elements

- **Multi-file skill packages** — workflow instructions + executable scripts + domain references (the Anthropic skill model)
- **Co-evolutionary verification** — generator + verifier co-evolve; verifier provides feedback without ground-truth
- **SkillsBench evaluation** — benchmark-validated on 87 tasks across 8 domains
- **Human–machine cognitive misalignment** — key problem addressed; human-curated skills can degrade agent performance in some domains (e.g., Natural Science)
- **Generalization testing** — validated on Claude Code + Codex + 6 additional LLMs

## How Context Elements Support Skills

The **Surrogate Verifier** is itself a novel context element — it acts as a formal quality gate for skill bundles, analogous to how [[schimatos-shacl-knowledge-graph|SHACL validation]] constrains knowledge graphs. Unlike static validation (SHACL shapes), the verifier co-evolves with the generator, creating a dynamic feedback loop.

The framework addresses a key finding from [[skillsbench-agent-skills-benchmark|SkillsBench]]: human-curated skills yield uneven gains due to cognitive misalignment. CoEvoSkills surpasses human-curated skills within 5 evolution iterations.

## Composition Notes

- **Tool–skill gap:** Existing self-evolving tool methods (Voyager, SkillCraft, Live-SWE-Agent) cannot generate multi-file skill packages
- **No ground-truth dependency:** Unlike EvoSkill (which requires ground-truth for failure diagnosis), CoEvoSkills operates without ground-truth test content
- **Multi-LLM generalization:** Skills generated by CoEvoSkills generalize across 8 LLMs
