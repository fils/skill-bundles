## What It Is

A bundle of **Claude Skills** (SKILL.md folders) specifically built for ontology and biocuration tasks. This is the most direct example of "skills + formal ontology context" we have found — the skills are designed to operate on OBO Foundry ontologies (GO, Mondo, Uberon, CL, EFO) using formal tooling (OAK, ROBOT, DOSDP).

## Context Elements — A Model Bundle

This is one of the cleanest examples of a **skills + formal context** bundle in our catalog:

- **Ontology (primary):** OBO Foundry ontologies (GO, Mondo, CL, Uberon, EFO, HPO) — formal, OWL-based, peer-reviewed
- **Validation/Rules:** OBO Foundry principles embedded in skills (e.g., `editing-obo-ontologies` enforces OBO conventions)
- **Tool Vocabulary:** OAK (Ontology Access Kit), ROBOT, OLS, BioPortal
- **Configuration schema:** `CLAUDE.md`, `AGENTS.md`, `ai.just` (Justfile for reproducibility)
- **Mapping:** Implicit — OAK supports cross-ontology term mapping (SSSOM-like)
- **Distribution:** `/plugin marketplace add ai4curation/curation-skills`

## Composition Notes

- **Auto-invocation pattern:** Skills are designed to be **auto-selected** based on user query, not explicitly invoked. Example: asking a question about GO terms triggers the OAK skill automatically.
- **MCP integration:** Composes with **OLS MCP** and **BioPortal MCP** for basic searches — OAK is reserved for complex operations.
- **GitHub Actions:** Bundled `.github/workflows/` configs let repositories run ontology-agent workflows (e.g., `ai-agent.yml` in Mondo).
- **Codex compatibility:** Canonical Claude location is `.claude/skills/`, but Codex reads from there too (cross-platform SKILL.md portability).
- **Example command generated by OAK skill:**
  ```bash
  runoak -i sqlite:obo:go info .in goslim_generic .and .desc//p=i GO:0008150
  ```
  Boolean queries combining graph and property lookups — a complex ontology operation made composable.

## Reproducibility Assessment

**High.** All skills are version-controlled in a public repo, use deterministic commands (OAK, ROBOT, OLS), and the reference implementations in Mondo/Cell Ontology provide working examples. The Justfile (`ai.just`) encodes repeatable workflows.

## Why It Matters for the Catalog

This bundle proves a thesis of the wiki: **skills that operate on formal artifacts (ontologies, rules) are more valuable than skills that don't.** The OAK skill is essentially a thin natural-language interface over a sophisticated ontology toolkit — and the OBO Foundry context (peer review, principled design, formal semantics) provides guardrails that prevent drift.

The `ai-blame` companion tool is interesting: it solves the "what did the agent actually change?" problem for ontology editing — a domain-specific provenance system that pairs with the editing skills.
