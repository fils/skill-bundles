# From Registry to Repository: How AI Agent Skills Are Written, Adapted, Maintained

**Source:** https://arxiv.org/abs/2607.00911 | https://arxiv.org/html/2607.00911v1
**Date:** Jul 1 2026 | Gao, Lulla, Lin, Baltes, Treude, Zahedi
**Signal rating:** 9/10

## Key ideas
- First empirical study of skills as **engineered artefacts** (not only capabilities).
- Corpus: 18,463 skills.sh registry + 23,199 personal GitHub skills (5,876 repos); 3,709 reuse links.
- SWEBOK classification: Software Construction dominates + long tail of specialized KAs.
- Content taxonomy (6 themes): scoping/orchestrating, execution lifecycle, output quality, governing agent conduct, grounding domain knowledge, user/agent coordination.
- Reuse is **one-time copy**: most near-verbatim; 53% never modified after adoption; maintenance additive; behavioral contracts almost untouched.
- Edit taxonomy (6): metadata, reworking operational specs, recalibrating constraints, adapting knowledge/resources, maintaining compatibility, polishing.

## Why skill-bundles
Establishes maintenance science for skill bundles: copy-fork drift, additive evolution, stable behavioral contracts — informs registry design and versioning priorities.
