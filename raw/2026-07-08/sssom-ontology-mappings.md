# SSSOM: Simple Standard for Sharing Ontology Mappings

**Source:** https://mapping-commons.github.io/sssom/dev/
**Date ingested:** 2026-07-08
**Key ideas:** Community standard for exchanging semantic entity mappings (mappings + mapping sets with metadata like provenance, confidence, justification). TSV format with YAML metadata header. Supports curation rules, author tracking. Used for ontology alignment, data integration across domains (biology, healthcare, IT).

**Quotes:**
- "SSSOM provides a standardized format to share mappings, enabling researchers and developers to more easily connect and utilize diverse datasets."
- "A **SSSOM mapping** is a statement that there is a correspondence between two semantic entities."
- Includes fields like mapping_justification, author_id, confidence, curation_rule.

**Relevance to skill bundles:** SSSOM mappings provide semantic context/layering for skills; bundles combining skills with SSSOM enable interoperable, mappable agent capabilities (e.g., cross-ontology skill reuse).
