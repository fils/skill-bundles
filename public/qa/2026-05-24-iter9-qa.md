# Q&A and Gap Analysis — 2026-05-24 Iteration 9

## Complex Queries Against Wiki Catalog

### Q1: Which bundles combine skills with both dependency graphs AND validation?
**A:** [graph of skills dependency retrieval](../examples/graph-of-skills-dependency-retrieval.md) provides dependency graph construction. [dspy agent skills bundle](../examples/dspy-agent-skills-bundle.md) has 80 validation tests. No single bundle yet combines both dependency-aware retrieval + programmatic skill validation. This is a gap: a bundle that validates both the correctness of individual skills AND their dependency compatibility before loading.

### Q2: What benchmarking coverage exists for skill bundles?
**A:** [skillsbench agent skills benchmark](../examples/skillsbench-agent-skills-benchmark.md) provides paired evaluation (with-skills vs without-skills delta). [awesome agent skills catalogs](../examples/awesome-agent-skills-catalogs.md) catalogs the market but doesn't benchmark it. Gap: no per-bundle benchmark — SkillsBench measures aggregate augmentation, not individual bundle effectiveness.

### Q3: Which bundles encode security posture as context elements?
**A:** [nvidia verified agent skills](../examples/nvidia-verified-agent-skills.md) (OpenSSF signing), [mondoo agent skills security](../examples/mondoo-agent-skills-security.md) (security analysis cataloging 14 attack patterns), [veto agent authorization](../examples/veto-agent-authorization.md) (authorization kernel). Mondoo's analysis identifies that 25%+ of public skills have vulnerabilities — verification bundles exist but coverage is thin for the 50k+ skills in registries.

### Q4: What ontology patterns appear most frequently alongside agent skills?
**A:**
- Dependency graphs (DAGs): 2 examples ([graph of skills dependency retrieval](../examples/graph-of-skills-dependency-retrieval.md), [graphguard os guardrails](../examples/graphguard-os-guardrails.md))
- Taxonomies: 3 examples ([claude skills ecosystem 1116](../examples/claude-skills-ecosystem-1116.md) MITRE/NIST mapping, [awesome agent skills catalogs](../examples/awesome-agent-skills-catalogs.md) platform matrix, [open ontologies github](../examples/open-ontologies-github.md) 32 stored ontologies)
- Attack taxonomies: 1 example ([mondoo agent skills security](../examples/mondoo-agent-skills-security.md) 14 patterns)

### Q5: How do successful bundles handle skill orchestration?
**A:** Two patterns found:
1. **Orchestrator skill**: [dspy agent skills bundle](../examples/dspy-agent-skills-bundle.md)'s `dspy-advanced-workflow` skill activates and sequences the other four skills
2. **Methodology cycle**: [superpowers engineering discipline skills](../examples/superpowers-engineering-discipline-skills.md)'s 6-stage linear workflow (Brainstorming → Git Worktree → Plan → Subagent → Review → Finishing)

## New Research Questions

1. **Dependency Resolution**: How does GoS's dependency graph relate to the `allowed-tools` frontmatter field? Can constraints be inferred from the DAG?
2. **Security-Validation Gap**: No bundle yet combines [nvidia verified agent skills](../examples/nvidia-verified-agent-skills.md) signing with [graph of skills dependency retrieval](../examples/graph-of-skills-dependency-retrieval.md) dependency checking. A bundle that verifies both authenticity and dependency compatibility before execution would address Mondoo's #1 recommendation.
3. **Benchmark-to-Bundle Mapping**: Can SkillsBench's paired evaluation be applied to individual bundles in our catalog? This would enable quantitative bundle quality claims.
4. **MCP vs Skills Complementarity**: Awesome-lists catalog both MCP servers and Skills. What bundles effectively combine both patterns? (Simon Willison argues Skills > MCP for simplicity, but the complementary pattern may exist.)
