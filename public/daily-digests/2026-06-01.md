## Key Patterns Surfaced

### 1. Three-Layer Validation Stack (NEW)
The skill ecosystem is converging on a three-layer validation architecture:
- **Layer 1 (Pre-commit):** skill-validator CLI — deep quality + contamination + LLM-as-judge
- **Layer 2 (CI/CD):** Validate Skill GitHub Action — structural spec compliance
- **Layer 3 (Registry):** agentskill.sh 12-category security scanner

This mirrors the **software supply chain** three-layer pattern (linters → CI scanners → registry audits) and is approximately 2-3 years behind the npm/PyPI ecosystem in maturity. Three independent projects (skill-validator, validate-skill, ags) emerged in 2026 to fill these three layers, suggesting the pattern is now stable.

### 2. Bidirectional SHACL ↔ LLM Bridge
With text2shacl (forward) added, the catalog now has a complete bridge:
- **Forward:** text2shacl turns natural-language requirements into SHACL shapes via multi-agent LangGraph pipeline with RAG
- **Reverse:** xpSHACL turns SHACL shapes into natural-language explanations

Together they form a closed loop: requirements → shapes → validation → explanation → refined requirements. This makes SHACL accessible to non-RDF-experts.

### 3. Ontology-Grounded Skill Bundles
Chris Mungall's `curation-skills` is the cleanest example in the catalog of skills operating on formal artifacts (OBO ontologies). The 7 skills (OAK, ROBOT, DOSDP, OBO editing, etc.) provide a natural-language interface to a sophisticated ontology toolkit. The OBO Foundry context (peer review, principled design, formal semantics) provides guardrails that prevent drift.

The `ai-blame` companion tool (line-level attribution for AI-assisted edits) is novel — it solves the "what did the agent actually change?" problem for ontology editing.

### 4. Convergence on Security Threat Taxonomy
Three independent projects ([nemoclaw security scanner](../examples/nemoclaw-security-scanner.md), [owasp agentic skills top 10](../examples/owasp-agentic-skills-top-10.md), [agentskill sh ags security scoring](../examples/agentskill-sh-ags-security-scoring.md)) all reference the same OpenClaw incident as motivation and have produced compatible threat taxonomies (10-class, 6-vector, 12-category — all cover the same core threats). The threat model is now well-defined.

## Gaps Identified

1. **Bundle-level validation:** All three validation layers target individual skills, not bundles — bundle-level interaction effects are an open problem.
2. **Attestation chaining:** The three validation layers produce independent signals — no SLSA-style provenance chain.
3. **Cross-platform consistency:** Layer 1 (skill-validator) supports 13 platforms; layers 2 and 3 have narrower support.
4. **Round-trip consistency for SHACL bridge:** No empirical test yet of "text → shapes → text" round-trip fidelity.
