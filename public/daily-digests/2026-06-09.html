Summary

Iteration 14 delivers a major expansion of domain-specific and infrastructure skill bundles, adding bioinformatics (ClawBio), production-grade engineering workflows (Addy Osmani), conference participation infrastructure (Agents at Conferences), and an updated DSPy optimization bundle (v0.2.3). Two foundational papers on autonomous scientific workflows (SC25) and the scAInce paradigm were also ingested as seeded references.

Key convergence signals:

  1. Domain specialization is maturing — ClawBio shows how 88 skills cohere around a single domain (genomics) with formal specification layers, reproducibility bundles, and a reference genome
  2. Process-as-skill bundles are emerging — Addy Osmani's 23 skills encode the entire SDLC with anti-rationalization design, verification gates, and Google engineering culture
  3. Infrastructure-as-skills — Agents at Conferences defines 6 core infrastructure components (registration, feeds, communication, moderation, benchmarking, memory) as agent capabilities
  4. Framework-specific bundles deepen — DSPy v0.2.3 adds exact version pinning, 34+ validators, GEPA gotcha documentation, and committed performance benchmarks

New Examples Added

1. ClawBio: Bioinformatics-Native Skill Library (clawbio bioinformatics skills)

  • 88 skills (29 production-ready) + 8,000+ Galaxy tools via Galaxy Bridge
  • Specification-First: Machine-readable SKILL.md contracts prevent hallucination in critical domains (dosage, genetics)
  • Reproducibility bundles: Every execution generates commands.sh + environment.yml + checksums.sha256
  • Reference genome (Corpasome): Real human genome (DOI: 10.5281/zenodo.19297389) for deterministic testing
  • Validation leaderboard: 92.3% (168/182) tests passing — public quantitative quality metric
  • Multi-modal interfaces: CLI, Telegram/Discord (RoboTerri), web (OpenClaw), Claude Code

2. Addy Osmani Agent Skills (addyosmani engineering skills)

  • 23 skills spanning full SDLC: Define → Plan → Build → Verify → Ship → Maintain
  • Anti-rationalization design: Explicit tables of agent excuses + required rebuttals
  • Verification-first: Non-negotiable evidence (tests, logs, runtime data) — "Seems right" never accepted
  • 7 slash commands mapping 1:1 to phases (/spec, /plan, /build, /test, /review, /code-simplify, /ship)
  • 3 specialist personas: code-reviewer, test-engineer, security-auditor
  • Reference checklists: Testing, Security, Performance, Accessibility
  • Google engineering culture: Hyrum's Law, Chesterton's Fence, Trunk-based development as normative artifacts

3. Agents at Conferences Infrastructure (agents at conferences infrastructure)

  • 6 core infrastructure components as agent capabilities:
    1. Registration API — Identity + capability schema (capabilities[], model_backbone)
    2. Conference Feed API — Real-time transcripts (30s updates), WebSocket live stream
    3. Communication Bus — Channel-based, 5 msg/min rate limit, structured format
    4. Moderation Layer — Human-in-the-loop dashboard, Q&A approval, spam filtering
    5. Benchmark Arena — Live GPQA/AAAR/LAB-Bench competition, real-time leaderboard
    6. Output Aggregation — PDF report + JSON API for inter-agent querying
  • Capability schema as skill taxonomy: capabilities[] array defines what skills an agent has
  • Protocol contracts: WebSocket + REST specs in ARCHITECTURE.md
  • Target: 50 agents at SciFM26 (May 27-29, 2026), TPC26 at SC26

4. DSPy Agent Skills v0.2.3 Update (dspy agent skills bundle)

  • Exact DSPy 3.2.1 pinning via check_dspy_surface.py — prevents API drift
  • 34+ pytest validators for SKILL.md frontmatter, JSON schemas, Python AST
  • GEPA gotchas documented: reflection_lm at construction time, Pydantic cloudpickle, minibatch sizing
  • Committed performance benchmarks: +19.53pp RAG, +8.33pp math, +0.098 invoice extraction
  • Offline validation: example_*.py --dry-run smoke tests
  • Dual platform: Claude Code + Codex CLI via marketplace and npx skills

The (R)evolution of Scientific Workflows (SC '25) (sc25 autonomous science workflows)

  • Two-dimensional evolution matrix: Intelligence (Static→Adaptive→Learning→Optimizing→Intelligent) × Composition (Single→Pipeline→Hierarchical→Mesh→Swarm)
  • State machine abstraction as common ground for workflows + agents
  • 100x discovery potential by reducing human bottlenecks
  • Intelligence Service Layer architecture for hypothesis, design, analysis agents

The Beginning of scAInce (scaince ai lab automation)

  • Co-pilot → Lab-pilot transition: AI acts on knowledge via self-driving laboratories (SDLs)
  • SDLs as ultimate skill bundles: ChemCrow (LLM + chemistry tools + robots), autonomous synthesis of 29 compounds
  • Information-gain scoring for experiment prioritization (Bayesian optimization)
  • Governance frameworks: EU AI Act, ISO 42001, PRISMA-AI, STARD-AI checklists
  • Machine-readable publications + FAIR data + Megasets as infrastructure requirements
