## Key Discoveries
- **addyosmani/agent-skills**: Production-grade 24-skill engineering lifecycle bundle (Define/Plan/Build/Verify/Review/Ship) with 4 personas, 6 reference checklists (testing, security, performance, etc.). Explicit verification gates, rationalization tables, and cross-platform support. Highest-signal bundle seen — embeds Google engineering culture as reusable skills + rules/taxonomies.
- **agentskills.io**: Authoritative open standard home. Defines SKILL.md + references/scripts/assets structure. Progressive disclosure model. Multi-agent support (Claude Code, Copilot, Codex, Gemini, Cursor, VS Code). Foundation for all future bundles.
- **heilcheng/awesome-agent-skills**: 1000+ curated real skills from official sources (Anthropic, OpenAI, Google, Vercel, Stripe, etc.). Quality standards, installation guides per platform, npx CLI. Demonstrates ecosystem scale and bundle patterns in the wild.
- **mapping-commons/sssom**: Canonical formal context element (ontology mapping TSV + 30+ metadata fields including curation_rule, mapping_justification). Perfect exemplar for bundling with semantic/agent skills. Active 2026 development. Bridges to SHACL/validation patterns.

## Emerging Patterns
- Lifecycle taxonomies (Define→Ship) are the dominant organizational structure in mature bundles.
- Verification/Red Flags/Rationalizations tables function as embedded rules engines.
- Official spec + community awesome lists + production examples form a self-reinforcing ecosystem.
- Formal standards like SSSOM are ready for integration as "context element" layers but adoption in agent-skills bundles is still emerging (gap to close).

## Gaps Identified
